# 用户卡片交互界面方案文档

## 1. 需求概述

本方案旨在构建一个基于 uni-app 的用户卡片交互界面，支持左右滑动匹配、照片轮播展示、筛选条件设置等核心功能。界面设计遵循现代 UI/UX 标准，确保视觉层次分明、色彩协调、交互流畅。

---

## 2. 功能说明

### 2.1 数据获取与处理

| 功能点 | 描述 | 技术实现 |
|--------|------|----------|
| 会员数据加载 | 从 API 获取会员列表，包含完整字段 | `GET /api/discover/recommend` |
| 照片资源处理 | 最多展示4张照片，按上传时间倒序 | 前端 slice(0, 4) 处理 |
| 默认图片处理 | 无照片时显示占位图 | 使用 SVG 默认图片 |
| 图片加载失败处理 | 图片加载失败时显示默认图 | `@error` 事件处理 |

**数据结构定义**：

```typescript
interface CardPhoto {
  id?: number
  url: string                    // 原图地址
  thumbnail_url?: string         // 缩略图地址
  is_main?: number               // 是否主图
  is_premium?: boolean           // 是否付费
  points_price?: number          // 积分价格
}

interface CardUser {
  id: number                     // 用户ID
  nickname: string               // 昵称
  avatar: string                 // 头像URL
  gender: string                 // 性别: 'male' | 'female'
  age: number                    // 年龄
  height: string                 // 身高(cm)
  weight: string                 // 体重(kg)
  hobbies: string                // 爱好
  love_declaration: string       // 个人宣言
  is_vip: number                 // 是否VIP
  distance?: string              // 距离
  location?: string              // 位置
  photos: CardPhoto[]            // 照片列表
}
```

### 2.2 卡片交互功能

| 交互类型 | 触发方式 | 效果 | 数据行为 |
|----------|----------|------|----------|
| Like | 右划 / 点击关注按钮 | 卡片滑出右侧，显示绿色 LIKE 标识 | 记录喜欢，自动关注，移除卡片 |
| Nope | 左划 / 点击跳过按钮 | 卡片滑出左侧，显示红色 NOPE 标识 | 记录不喜欢，降低匹配权重，移除卡片 |

**滑动参数配置**：

| 参数 | 值 | 说明 |
|------|-----|------|
| 滑动阈值 | 100px | 超过此距离触发滑动操作 |
| 旋转系数 | 0.05 | 滑动距离 × 系数 = 旋转角度 |
| 动画时长 | 200ms | 卡片滑出动画时间 |
| 回弹动画 | 300ms | 未达到阈值时的回弹时间 |

**按钮功能映射**：

| 按钮 | 功能 | 状态控制 |
|------|------|----------|

| 跳过 | 等同于左划（踩+降低权重） | 始终可用 |
| 聊天 | 跳转聊天页面 | 未关注时置灰不可用 |
| 查看 | 展示会员详细信息 | 始终可用 |

**筛选面板结构**：

**筛选条件配置**：

| 条件 | 类型 | 范围 | 步长 | 默认值 |
|------|------|------|------|--------|
| 性别 | 单选 | 不限/男/女 | - | 不限 |
| 身高 | 范围 | 140-190cm | 5cm | 140-190 |
| 体重 | 范围 | 40-100kg | 5kg | 40-100 |

### 2.5 数据展示规则

- **好友过滤**：已成为好友的会员不再出现在卡片列表中
- **实时更新**：好友状态变更后，列表立即刷新
- **分页加载**：卡片列表少于3张时自动加载更多

## 4. 交互流程图

### 4.1 卡片滑动交互流程

```
用户触摸卡片
    │
    ▼
记录起始坐标 (touchstart)
    │
    ▼
用户拖动卡片
    │
    ▼
实时更新位移和旋转角度 (touchmove)
    │
    ├─ deltaX > 0 → 显示 LIKE 标识（绿色）
    │
    └─ deltaX < 0 → 显示 NOPE 标识（红色）
    │
    ▼
用户释放卡片 (touchend)
    │
    ├─ |deltaX| > 100px
    │       │
    │       ├─ deltaX > 0 → 滑出右侧 → 触发 Like 操作
    │       │
    │       └─ deltaX < 0 → 滑出左侧 → 触发 Nope 操作
    │
    └─ |deltaX| ≤ 100px
            │
            ▼
        回弹到原始位置
```

### 4.2 Like 操作流程

```
触发 Like
    │
    ▼
发送 API 请求 POST /api/discover/like/{userId}
    │
    ▼
┌─────────────────────┐
│  是否匹配成功？      │
└─────────────────────┘
    │
    ├─ 是 → 显示匹配成功弹窗
    │       │
    │       ├─ 点击「立即聊天」→ 跳转聊天页面
    │       │
    │       └─ 点击「稍后再聊」→ 关闭弹窗
    │
    └─ 否 → 显示「已喜欢」提示
    │
    ▼
从卡片列表中移除该用户
    │
    ▼
卡片数量 < 3？
    │
    └─ 是 → 加载更多用户
```

### 4.3 筛选操作流程

```
点击筛选按钮
    │
    ▼
弹出筛选面板（从底部滑入）
    │
    ▼
用户设置筛选条件
    │
    ├─ 性别选择
    │
    ├─ 身高范围调整
    │
    ├─ 体重范围调整
    │
    └─ 年龄范围调整
    │
    ▼
┌─────────────────────┐
│  点击「重置」或「确定」│
└─────────────────────┘
    │
    ├─ 重置 → 恢复默认条件
    │
    └─ 确定 → 应用筛选条件
            │
            ▼
关闭筛选面板
    │
    ▼
刷新会员列表（带筛选参数）
```

```
    ┌──────────────┐      ┌──────────────┐

          ▼
    ┌──────────────┐
    │   移除状态    │

```

**状态转换条件**：

| 当前状态 | 触发条件 | 目标状态 |
|----------|----------|----------|
| Idle | touchstart | Dragging |
| Dragging | touchmove | Dragging（持续更新） |
| Dragging | touchend + \|deltaX\| > 100 | SwipeOut |
| Dragging | touchend + \|deltaX\| ≤ 100 | Bounce |
| Bounce | 动画完成 | Idle |
| SwipeOut | 动画完成 | Removed |

## 6. 异常场景处理

### 6.1 数据加载异常

| 场景 | 处理方式 | 用户提示 |
|------|----------|----------|
| 网络请求失败 | 显示加载失败状态，提供重试按钮 | "加载失败，请重试" |
| 服务器错误 | 显示错误提示，延迟后自动重试 | "服务器繁忙，请稍后再试" |
| 无数据返回 | 显示空状态提示 | "暂时没有更多用户" |
| Token 过期 | 自动跳转到登录页面 | 无（直接跳转） |

### 6.2 图片加载异常

| 场景 | 处理方式 | 用户提示 |
|------|----------|----------|
| 图片 URL 无效 | 使用默认占位图片 | 无（静默处理） |
| 图片加载失败 | 使用默认占位图片 | 无（静默处理） |
| 网络超时 | 使用默认占位图片 | 无（静默处理） |

### 6.3 操作异常

| 场景 | 处理方式 | 用户提示 |
|------|----------|----------|
| 重复操作 | 禁用按钮，防止重复提交 | "操作中，请稍候" |
| API 返回错误 | 显示错误提示 | 根据错误类型显示对应提示 |
| 匹配成功后用户不存在 | 关闭弹窗，刷新列表 | "用户不存在" |

## 8. 性能优化策略

### 8.1 图片优化

- 使用缩略图 URL 加载卡片图片
- 图片懒加载，只加载当前可见卡片的图片
- 使用 `mode="aspectFill"` 确保图片正确裁剪

### 8.2 渲染优化

- 限制同时渲染的卡片数量（最多3张）
- 使用 `will-change: transform` 优化动画性能
- 使用 CSS3 硬件加速（`transform: translate3d`）

### 8.3 数据缓存

- 缓存已加载的用户数据
- 缓存用户互动状态（Like/Nope）
- 使用本地存储保存筛选条件
