# 环信用户同步命令使用说明

## 命令功能

此命令用于将数据库中的用户批量同步到环信平台。

## 使用方法

### 基本用法（只注册新用户）

```bash
php artisan easemob:sync-users
```

此模式会：
- 检查每个用户是否已在环信存在
- 只注册不存在的用户
- 跳过已存在的用户

### 强制模式（更新所有用户）

```bash
php artisan easemob:sync-users --force
```

此模式会：
- 更新所有用户的昵称信息
- 适用于批量更新用户信息
- 会覆盖环信上的昵称

## 使用场景

### 1. 初次部署
首次部署时，数据库已有用户，使用：
```bash
php artisan easemob:sync-users
```

### 2. 批量更新昵称
如果用户昵称需要批量同步：
```bash
php artisan easemob:sync-users --force
```

### 3. 定期检查
可以添加到定时任务中定期检查：
```bash
# crontab 示例（每天凌晨 2 点执行）
0 2 * * * cd /path/to/project && php artisan easemob:sync-users
```

## 日志记录

所有操作都会记录到 Laravel 日志中：
- 成功：`storage/logs/laravel.log` 中记录 "批量同步用户到环信成功"
- 失败：记录 "批量同步用户到环信失败" 及错误信息

查看实时日志：
```bash
tail -f storage/logs/laravel.log
```

## 注意事项

1. **不会影响现有功能**：同步命令只注册不存在的用户，不影响已存在的用户
2. **失败不影响流程**：单个用户失败不会影响其他用户的同步
3. **自动重试**：可以多次运行命令，已存在的用户会被跳过
4. **性能考虑**：大量用户时可能需要一些时间，进度条会显示进度

## 相关文件

- 命令文件：`app/Console/Commands/SyncUsersToEasemob.php`
- Observer: `app/Observers/UserObserver.php`
- 服务类：`app/Services/EasemobService.php`

## 自动同步机制

除了手动运行命令外，系统还实现了自动同步：

1. **新用户注册**：Observer 自动注册到环信
2. **用户更新**：Observer 自动更新环信昵称
3. **用户删除**：Observer 自动删除环信账户

因此，同步命令主要用于：
- 初始化已有用户
- 批量修复数据
- 定期同步检查
